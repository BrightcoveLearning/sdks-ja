概要

Brightcoveのネイティブ SDK には、コンテンツの収益化に役立つ一連のオプションが用意されています。Native SDKとそれらの統合によって、プレロール、ミッドロール、ポストロール広告やコンパニオン広告などの機能を使って、プレーヤーの広告を最適化する方法をパブリッシャーは決定できます。

注意:

  • Brightcoveのネイティブ SDK は、VPAYD 広告をサポートしていません。
  • Androidの場合MediaPlayer、HLS動画の再生中にミッドロール広告を使用することには既知の制限があります。Brightcoveのネイティブプレーヤー SDK を使用すると、これらの問題の多くを解決できます。Android 用ネイティブ SDK で HLS 再生について詳しく説明します。

キューポイント

広告 URL は、広告キューポイントを使用して、または VMAP 広告タグ URL 内で定義できます。

  • 広告キューポイントでは、それぞれに VAST 広告タグ URL を設定する必要があります。
  • 広告キューポイントを使用しない場合、VMAP 広告タグ URL を使用して、広告がいつ再生されるかを定義できます。
VMAP URLは広告の再生リストであり、VASTURLは個々の広告を定義します。一般的な情報については、次の学習ガイドを参照してください。ビデオ広告資料。

クライアント側のプラグイン

クライアントサイド広告は、クライアントデバイスで広告要求と広告挿入を行う従来の方法です。この方法では、特定の広告キューポイントにヒットするまでメイン動画が再生されます。このポイントで、構成された広告サーバーに要求が行われ、広告サーバーは適切な広告詳細で応答します。すると、プレーヤーは要求された広告の再生を始めます。

Brightcove統合は広告要求の実行、広告詳細の解釈、広告の表示といった作業を処理することによって、このプロセスを簡単にします。

グーグルのインタラクティブメディア広告 (IMA)

Brightcoveは Google DoubleClick とパートナーとなり、Webプラットフォーム（Flash/HTML5）とモバイルプラットフォーム（iOS/Android）の両方で GoogleのIMA SDK のサポートを開発した、最初のオンライン動画プラットフォームでした。Googleと弊社との密接な関係によって、弊社は近く公開される機能やバグ修正の最新情報を入手できます。

パブリッシャーはBrightcove IMA SDKを使用して、DoubleClick for Publishers（DFP）、Google AdSenseネットワークが提供する広告の要求と表示ができます。また、Brightcove IMA SDKが解釈するVMAP標準に準拠していれば、このSDKを使用して、自身のVMAPプレイリストを解析することもできます。

iOSとAndroidの両方でGoogleのIMA SDKを使用して、両方のプラットフォームで使いやすい統合を提供します。これにより、Googleの更新情報を最新の状態に保つことができます。

サーバー側広告挿入 (SSAI)

サーバーサイドの広告スティッチングは、視聴者に広告を配信するための革新的な方法です。単一の連続的なストリームを提供しながら、広告がコンテンツに埋め込まれます。これによって広告要求が不要になり、ロード時間も短縮されます。また、視聴者は広告をブロックできません。Brightcove Native Player SDKは、広告とコンテンツとの切り替えでの再生エクスペリエンスがこれまでよりスムーズになるよう、適切な論理および広告UIを提供します。
