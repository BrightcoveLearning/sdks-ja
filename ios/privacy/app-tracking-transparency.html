はじめに

iOS 14.5には、App TrackingTransparencyと呼ばれる新しいプライバシー機能があります。アプリでこの機能を使用して、ユーザーに追跡許可をリクエストできます。ユーザーが追跡にノーと言うと、アプリはAppleの広告主ID（IDFA）やハッシュ化されたメールアドレスなどの他の識別子を使用して、広告ターゲティングのアクティビティに関するデータを共有できなくなります。

アプリがエンドユーザーデータを収集し、アプリやWebサイト全体で追跡する目的で他の企業と共有する場合は、AppTrackingTransparencyフレームワーク。

詳細については、以下を参照してください。

  • App Store：ユーザーのプライバシーとデータの使用
  • アプリ追跡の透明性
  • IMA SDK：SKAdNetworkを有効にしてコンバージョンを追跡する
  • INVIDIパルスドキュメンテーション
iOSおよびtvOS用のネイティブSDKは、 BrightcoveIMAおよびPulseプラグイン。

要件

アプリトラッキングの透明性には、次の要件が必要です。

プラットフォーム

システム要件は次のとおりです。

  • iOSまたはtvOSバージョン14.5以降

BrightcoveネイティブSDK

ネイティブSDKの最新バージョンを使用することをお勧めします。

  • iOSまたはtvOSバージョン6.8.0以降のネイティブSDK
  • 詳細については、 SDKリリースノート

実装

以下に示すサンプルアプリは、アプリが広告主のID（IDFA）を使用できるかどうかを判断するために、App TrackingTransparencyアラートを表示する方法を示しています。

  1. 次のサンプルアプリのいずれかを開いて確認します。

    • BasicIMAPlayer
    • BasicPulsePlayer
  2. を開きますBaseViewController、およびを見つけますviewDidLoad方法。

  3. iOS 14の場合、SDKはの値をチェックしますATTrackingManager.trackingAuthorizationStatus。値がATTrackingManagerAuthorizationStatusAuthorizedの場合、SDKはIDFAを使用します。

    if (@available(iOS 14, *))
  {
  __weak typeof(self) weakSelf = self;
  [ATTrackingManager requestTrackingAuthorizationWithCompletionHandler:^(ATTrackingManagerAuthorizationStatus status) {
      __strong typeof(weakSelf) strongSelf = weakSelf;
      dispatch_async(dispatch_get_main_queue(), ^{
          // Tracking authorization completed. Start loading ads here.
          [strongSelf videoLibrary];
          
          [strongSelf setupPlayerView];
          [strongSelf setupPlaybackController];
          [strongSelf requestVideo];
      });
  }];
  4. 古いiOSバージョンの場合、SDKは次の値を使用しますASIdentifierManager.sharedManager.isAdvertisingTrackingEnabled IDFAを使用する必要があるかどうかを判断します。
