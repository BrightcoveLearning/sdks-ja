はじめに

デバッグに役立つデフォルトのログメッセージを使用してビデオアプリをテストします。アプリを公開する準備ができたら、これらのログメッセージを削除できます。

ログメッセージの削除

Android 版 Native SDK からデフォルトのログメッセージを削除するには、次の手順を実行します。

  1. アプリで、 proguard-rules.proファイル。
     
    Android Studioでは、このファイルはデフォルトで作成されます。このファイルがBrightcoveAndroidサンプルのどこにあるのか疑問に思われる場合は、このファイルの名前を次のように変更しました。 proguard-project.txt。
  2. 以下のコードブロックを追加して、アプリからログを削除します。 
    -assumenosideeffects class android.util.Log {
  public static *** v(...);
  public static *** d(...);
  public static *** i(...);
  public static *** w(...);
  public static *** e(...);
}
    ログメッセージの詳細については、 Androidログクラスドキュメンテーション。
  3. 次に、アプリのbuild.gradleファイルを開きます。
  4. buildTypesセクションで、minifyEnabledの値を持つ属性を追加しますtrue。
  5. 前のコードの下に、proguardFiles次のように属性を追加します。
    • getDefaultProguardFile()メソッドを使用して、Android SDK から proguard-android-optimize.txt ファイルを取得します。
    • proguard-rules.pro ファイルを参照します。

    あなたのbuild.gradleファイルは次のようになります。 
    android {
    compileSdkVersion 25
    buildToolsVersion '25.0.0'

    defaultConfig {
        ...
    }

    buildTypes {
        release {
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'),
            'proguard-rules.pro'
        }
    }
}
Android Studioは、インスタント実行の使用中にJavaコードカバレッジライブラリ（JaCoCo）とProGuardを一時的に無効にします。インスタント実行はデバッグビルドでのみ動作するため、これはリリースビルド ( https://developer.android.com/studio/run/index.html ) には影響しません。

デバッグビルドタイプのログを削除する場合は、次のことを行う必要があります。インスタント実行を無効にする。
